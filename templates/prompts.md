# 提示词模板

## 新项目启动

```text
我要开始一个新项目：{项目名}

请你创建这个项目的工作现场，并先问清楚最少必要信息：
1. 这个项目要完成什么
2. 当前已经有什么材料
3. 最近一步要做什么
4. 什么结果算做完

然后生成项目目录和基础文件。
```

## 材料入库

```text
请把下面这份材料放进对应项目的工作现场。

你需要先判断：
1. 它属于哪个项目
2. 它是什么类型的材料
3. 原文应该保存在哪里
4. 素材索引应该怎么写
5. 是否产生新的判断、待办或风险

材料如下：
{材料}
```

## 开工调度

```text
我要继续处理这个任务：{任务}

请你先不要直接执行。

先读取对应项目的工作现场，输出：
1. 当前识别项目
2. 当前阶段
3. 本次任务应该继承哪些判断
4. 哪些方向不能再走
5. 本次完成标准
6. 需要使用哪些素材或方法

确认后再开始执行。
```

期望输出：

```text
识别项目：
置信度：
命中证据：
当前阶段：
继承判断：
不能再走：
完成标准：
待读材料：
下一步动作：
```

## 任务结束回写

```text
这次任务已经完成。

请你判断哪些内容需要回写到项目工作现场：
1. 项目进度变化
2. 新确认判断
3. 新否定方向
4. 新素材
5. 新产出
6. 新待办
7. 可复用方法
8. 本次踩坑

不要把临时想法写成长期规则。
不要把项目私有判断写成全局方法，除非它可以跨项目复用。
```

期望输出：

```text
拟回写：
- 文件：
- 类型：
- 内容摘要：
- 证据来源：
- 是否覆盖：

不回写：
- 原因：

需要用户确认：
```
